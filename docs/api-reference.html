🔐 Authentication

POST /auth/login Authenticate and get JWT tokens
Request Body
emailstringrequiredUser email address
passwordstringrequiredAccount password
Response
{ "success": true, "data": {
  "user": { "id", "email", "role", "fullName", "profileId" },
  "accessToken":  "eyJ...",   // expires in 8h
  "refreshToken": "eyJ..."    // expires in 7d
} }
POST /auth/refresh Get new access token using refresh token
Request Body
refreshTokenstringrequiredRefresh token from login
Response
{ "success": true, "data": { "accessToken": "eyJ..." } }
GET /auth/me 🔒 any role Get current user profile 
{ "success": true, "data": { "id", "email", "role", "fullName", "phone", "lastLoginAt" } }
PUT /auth/change-password 🔒 any role
Request Body
currentPasswordstringrequired
newPasswordstringrequiredMin 8 chars, must contain a number
POST /auth/users admin Create teacher or student account
Request Body
emailstringrequired
passwordstringrequired
roleenumrequiredadmin | teacher | student
fullNamestringrequired
phonestringoptional

📅 Classes (scheduling core)

GET/classesadminList classes with filters
Query Params
datedateoptionalYYYY-MM-DD for daily view
from / todateoptionalDate range
teacher_iduuidoptional
course_iduuidoptional
statusenumoptionalscheduled|ongoing|completed|cancelled
POST /classes admin Create class (single or recurring) — runs conflict check
Request Body
teacherIduuidrequired
subjectIduuidrequired
courseIduuidoptional
scheduledDatedaterequired
startTimetimerequiredHH:MM format
durationMinutesnumberrequired40 | 60 | custom
platformenumrequiredskype|google_meet|microsoft_teams|moodle|custom
meetingLinkstringoptional
studentIdsuuid[]optionalIndividual students
groupIduuidoptionalAssign whole group
recurrenceobjectoptionalSee recurrence object below
Recurrence Object (optional)
{
  "frequency":   "weekly",           // daily | weekly | monthly
  "daysOfWeek":  ["monday","wednesday","friday"],
  "endDate":     "2025-06-30",       // or use totalSessions
  "totalSessions": 24
}
Conflict Response (409)
{ "success": false, "message": "Scheduling conflict detected.",
  "conflicts": [
    { "type": "teacher_overlap", "detail": "Teacher has a class 10:00–11:00" },
    { "type": "daily_limit",     "detail": "Would exceed 8h daily limit (7.5h scheduled)" }
  ]
}
PUT/classes/:idadminReschedule or update a class — re-runs conflict check
Same fields as POST. Original class is preserved with status "rescheduled".
DELETE/classes/:idadminCancel a class
reasonstringrequired
